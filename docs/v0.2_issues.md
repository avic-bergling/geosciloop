# GeoSciLoop v0.2 Issues

This issue list breaks v0.2 into implementation tasks for the fixture-based real-data adapter prototype.

## 1. DataRequest Schema

- Add `AOI`, `TimeRange`, `DataSourceRequest`, and `SplitStrategy` dataclasses.
- Support nested dry-run config fields while preserving v0.1 config compatibility.
- Validate required dry-run fields without forcing them onto v0.1 configs.

## 2. DataSourceManifest Schema

- Add `DataSourceManifest` dataclass with schema version, project name, mode, offline flag, dry-run flag, sources, and split strategy.
- Add serialization helpers compatible with existing JSON artifact writer.

## 3. Base DataAdapter Protocol/Interface

- Add `geosciloop/adapters/base.py`.
- Define `AdapterPlan`, `FetchResult`, and `DataAdapter` protocol.
- Keep all interface methods deterministic and side-effect-light.

## 4. Fixture-Backed STAC Adapter

- Add `geosciloop/adapters/stac_adapter.py`.
- Load Landsat and Sentinel fixture items by request role or collection.
- Return realistic STAC-style metadata with mock hrefs.
- Do not call a live STAC catalog in required tests.

## 5. Fixture-Backed GEE Adapter

- Add `geosciloop/adapters/gee_adapter.py`.
- Return dry-run collection metadata from a fixture.
- Mark live Earth Engine access as future work.
- Do not authenticate or require credentials.

## 6. Fixture-Backed OSM Adapter

- Add `geosciloop/adapters/osm_adapter.py`.
- Load roads metadata from an Overpass-like fixture.
- Return source records with vector role metadata.
- Do not call Overpass in required tests.

## 7. Provenance Metadata Support

- Extend or reuse provenance helpers so `DataSourceRecord` entries include provider, collection/dataset, href, query, license, and provenance metadata.
- Redact secrets in recorded query metadata.

## 8. Dry-Run Workflow

- Add a dedicated workflow module for real-data planning dry runs.
- Generate adapter plan, source manifest, validation report, ledger, report, summary, benchmark summary, and run log.
- Ensure `download_performed` is always false for the template.

## 9. Metadata Validators

- Add CRS, resolution, alignment, NoData, cloud/shadow QA, provenance, and split-strategy validators.
- Emit existing `ValidationResult` objects.
- Preserve warnings in reports and ledger.

## 10. Report/Summary Generation

- Add dry-run report builder.
- Add dry-run summary builder.
- Include dry-run disclaimer, planned source details, warnings, limitations, and live-run prerequisites.

## 11. Benchmark/Evaluator Extension

- Add dry-run evaluator sub-scores.
- Keep v0.1 evaluator behavior unchanged.
- Add tests for sub-score presence and failure details.

## 12. CLI/Config Support

- Add `--dry-run` to `geosciloop run`.
- Dispatch `mode: dry_run` or CLI dry-run override to the dry-run workflow.
- Preserve `--offline` for v0.1 demos.

## 13. Tests

- Add schema tests for dry-run config fields.
- Add adapter fixture tests for STAC, GEE, OSM, and population metadata.
- Add metadata validator tests.
- Add workflow and CLI dry-run tests.
- Add ledger, summary, report, benchmark, and gitignore checks.
- Re-run all v0.1 tests.

## 14. Docs

- Update README, CHANGELOG, roadmap, decisions, adapter docs, data-source docs, validator docs, dry-run workflow docs, and v0.2 release-readiness docs.
- Clearly state that v0.2 is a fixture-based prototype and not a live real-data analysis system.

## 15. Release Audit

- Add `docs/v0.2_release_audit.md`.
- Record verification commands, results, generated artifacts, known warnings, limitations, readiness status, blockers before tagging, and recommended next step.
- Do not tag or push.
