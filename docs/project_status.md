# Project Status

Audit date: 2026-06-21

This inventory reflects the files found in the working tree during the milestone audit. It separates implemented code from documented plans and notes weak verification where applicable.

## Repository State

- The working tree contains v0.2 release-candidate implementation changes until they are committed.
- The base package version is `0.1.0` in `pyproject.toml`.
- The base dependency set remains offline-friendly: `matplotlib`, `numpy`, `pandas`, `PyYAML`, and `scikit-learn`; `pytest` is a development extra.
- Optional extras `geo` and `gee` exist but are empty.

## Package Structure

- `geosciloop.cli`: command-line entrypoint for `geosciloop run`.
- `geosciloop.core`: schema parsing, runner dispatch, workflow state, artifact writing, and research ledger.
- `geosciloop.agents`: deterministic planner, critic, and writer shims. These are not autonomous multi-agent runtimes.
- `geosciloop.workflows`: synthetic UHI workflow and v0.2 fixture-backed dry-run planning workflow.
- `geosciloop.tools`: synthetic data, spatial grid, spectral indices, modeling, and plotting helpers.
- `geosciloop.validators`: deterministic config, value range, missing value, model metric, encoding, spatial split, multicollinearity, risk-indicator, claim-support, and v0.2 metadata checks.
- `geosciloop.report`: Markdown report and human-readable summary writers.
- `geosciloop.benchmark`: offline decomposed evaluator and benchmark summary writer.
- `geosciloop.adapters`: optional STAC, Google Earth Engine, OpenStreetMap, provenance helpers, and v0.2 fixture-backed adapter interfaces.

## Available Configs

- `configs/uhi_synthetic_demo.yaml`: offline deterministic synthetic UHI demo.
- `configs/uhi_morphology_synthetic_demo.yaml`: offline deterministic morphology UHI demo with building morphology, functional zones, and exposure role separation.

All configs use `task_type: urban_heat_island`; the v0.2 template sets `mode: dry_run`, `offline: true`, and `dry_run: true`.

## Available Validators

- `check_config_schema`: required config fields.
- `check_value_ranges`: NDVI, NDWI, NDBI, LST, density fields, morphology fields, and functional-zone allowed values.
- `check_missing_values`: null detection.
- `check_model_metrics_exist`: finite model metrics and metric ranges.
- `check_categorical_encoding_documented`: functional-zone one-hot encoding metadata.
- `check_spatial_split_strategy`: warns on random split over x/y grid data.
- `check_risk_indicator_model_usage`: warns if risk indicators enter model features.
- `check_predictor_multicollinearity`: pairwise predictor correlation and building-height interpretation warnings.
- `check_conclusion_support`: warns on missing causal design and checks association-claim direction against evidence.
- v0.2 metadata validators: CRS, raster resolution, raster alignment, NoData, cloud/shadow QA metadata, source provenance, and split strategy.

Status: implemented and tested. The checks are deterministic and do not call LLMs or network services.

## Benchmark And Evaluator Components

- `geosciloop.benchmark.evaluator.evaluate_run`: computes decomposed offline sub-scores.
- `geosciloop.benchmark.evaluator.evaluate_dry_run`: computes v0.2 dry-run sub-scores for source manifests, metadata validation, provenance, adapter plans, ledger, reproducibility, disclaimers, and no-live-dependency checks.
- `geosciloop.benchmark.evaluator.write_benchmark_summary`: writes `benchmark_summary.json`.
- Evaluator checks artifact completeness, model metric availability, validation failures, report claim support, and reproducibility metadata. `benchmark_summary.json` is the source of truth for artifact completeness and benchmark-style quality-gate checks.
- Evaluator intentionally does not produce a single `ai_scientist_score`.

Status: implemented and tested for offline synthetic artifacts and v0.2 dry-run artifacts. It validates artifact presence and metadata completeness, not real-world scientific validity.

## Harness Decisions

- `AGENTS.md`, `docs/harness_strategy.md`, `docs/harness_evaluation.md`, and `docs/decisions.md` recommend a simple deterministic runner for v0.1.
- LangGraph is deferred until durable agent state, retries, or human-review loops are needed.
- Snakemake is deferred until real-data file DAGs and partial reruns justify it.
- Prefect is deferred until scheduled or production-style orchestration exists.

Status: documented and aligned with the current code. No external harness has been added.
