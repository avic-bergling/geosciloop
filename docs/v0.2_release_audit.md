## Offline Guarantee

- Required tests and demos use local synthetic data or local fixtures.
- The v0.2 dry-run uses `tests/fixtures/` and `mock://` hrefs.
- No required test downloads external datasets.
- No required workflow requires internet access.

## No-Live-Dependency Guarantee

- `data_source_manifest.json` records `downloaded: false` for every source.
- `data_source_manifest.json` records `requires_credentials: false` for every source.
- `research_ledger.json` records `dry_run: true`, `download_performed: false`, and `credentials_required: false`.
- `run_log.json` records `dry_run: true`, `download_performed: false`, and `credentials_required: false`.
- The v0.2 dry-run does not authenticate to Google Earth Engine, call live STAC catalogs, call Overpass, or download population grids.

## Known Warnings

The dry-run template emits one expected warning:

- `split_strategy`: random split is planned for a geospatial AOI, so spatial leakage may inflate future model performance.

This warning is intentional. Real geospatial modeling should use spatial or block splits.

## Known Limitations

- v0.2 is a fixture-based real-data adapter prototype, not a live real-data analysis system.
- Fixture metadata is representative and not authoritative provider metadata.
- No raster pixels, vector geometries, or population grid values are downloaded or processed.
- Metadata validators check planning completeness, not live data quality.
- No real-world UHI conclusions are made.
- GeoSciLoop is still not a complete autonomous scientist.
- Provider-backed access remains future work outside required offline tests.
- v0.3 harness decisions remain deferred.
